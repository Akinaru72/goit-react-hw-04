# goit-react-hw-04

## Загальні вимоги

- Створіть репозиторій з назвою **goit-react-hw-04**
- При здачі домашньої роботи надайте два посилання:
  - на репозиторій з вихідним кодом
  - на задеплоєну сторінку проєкту (Vercel)
- Проєкт повинен бути створений за допомогою [**Vite**](https://vite.dev/)
- Під час запуску коду в консолі не повинно бути помилок або попереджень
- Для кожного компонента у папці `src/components` повинна бути окрема папка, яка містить:
  - JSX-файл React-компонента
  - файл стилів
- Назви папки, компонента (`.jsx`) та файлу стилів (`.module.css`) повинні бути однаковими та відповідати вимогам завдання
- Для експорту компонентів використовуйте **export default**
- Для HTTP-запитів використовуйте бібліотеку **axios**
- Код повинен бути чистим та відформатованим за допомогою **Prettier**
- Стилізація виконується за допомогою **CSS-модулів**

## Сервіс пошуку зображень

У цьому завданні за допомогою HTTP-запитів ти будеш отримувати зображення з сервісу [**Unsplash**](https://unsplash.com/developers).

### Підготовка

1. Зареєструй обліковий запис розробника
2. Створи новий додаток (**New Application**)
3. Придумай назву (наприклад):
   - Image Gallery
   - Photo Search
4. Отримай **Access Key** у розділі **Keys**

> ⚠ Для безкоштовної версії доступно **50 запитів на годину**  
> Не роби зайвих запитів!

## Хедер з формою пошуку

Компонент **SearchBar** приймає проп:

- `onSubmit` — функція для передачі значення інпуту при сабміті

### Структура DOM

```html
<header>
  <form>
    <input
      type="text"
      autocomplete="off"
      autofocus
      placeholder="Search images and photos"
    />
    <button type="submit">Search</button>
  </form>
</header>
```

## Валідація поля пошуку

Якщо під час натискання кнопки відправки форми текстове поле порожнє:

- Покажи користувачеві сповіщення про те, що потрібно ввести текст для пошуку зображень
- Перевірка виконується в компоненті **SearchBar** у момент сабміту форми
- Для відображення сповіщень використовуй бібліотеку [**React Hot Toast**](https://react-hot-toast.com/)

# Галерея зображень

Компонент **ImageGallery** — це список карток зображень, який створює DOM-елемент наступної структури:

```html
<ul>
  {/* Набір елементів списку із зображеннями */}
  <li>
    <div>
      <img src="" alt="" />
    </div>
  </li>
</ul>
```

## Умови рендерингу галереї

- Галерея повинна рендеритися **лише тоді**, коли є завантажені зображення
- Це добра практика **не включати елемент `<li>`** в компонент картки
- Елемент `<li>` має залишатися частиною компонента **ImageGallery**

# Картка зображення

Компонент **ImageCard** рендериться всередині галереї  
та створює DOM-елемент наступної структури:

```html
<div>
  <img src="" alt="" />
</div>
```

# Індикатор завантаження

Компонент **Loader** відображається під галереєю під час завантаження зображень.

- Можна використовувати будь-який готовий компонент, наприклад [**react-spinners**](https://www.npmjs.com/package/react-spinners) або інший
- Поки триває завантаження, індикатор **не повинен замінювати галерею**
- Loader має просто рендеритися **під галереєю**

> ⚠ Це критично важливо при підвантаженні нових зображень до вже існуючих

# Повідомлення про помилку

Компонент **ErrorMessage**:

- Рендериться **замість галереї** у разі помилки HTTP-запиту
- Достатньо реалізувати у вигляді **текстового повідомлення**

# Кнопка завантаження додаткових зображень

Компонент **LoadMoreBtn**:

- Рендерить кнопку з текстом **"Load more"**
- При кліку:
  - завантажується наступна порція зображень
  - нові зображення додаються до вже існуючих

### Умови рендерингу

- Кнопка відображається **лише якщо є завантажені зображення**
- Якщо масив зображень порожній — кнопка **не рендериться**

---

# Модальне вікно

Компонент **ImageModal**:

- Рендериться всередині компонента **App**
- Отримує всі необхідні дані та функції через **props**

### Поведінка

- При кліку на зображення відкривається модальне вікно
- Фон модального вікна — затемнений
- Відображається зображення у **великому форматі**
- Модальне вікно закривається:
  - при натисканні клавіші **ESC**
  - при кліку **поза межами** вікна

Для реалізації використовуй бібліотеку [**React Modal**](https://github.com/reactjs/react-modal?tab=readme-ov-file#examples)
